### 第 1 分钟：了解这是什么

**问题**: 每次修改数据库结构都要手动编写 SQL？

**解决**: 用 Python 代码定义数据库结构，自动同步到 PostgreSQL！

```python
# 就像这样定义表
class City(Base):
    __tablename__ = "cities"
    city_id = Column(String(50), primary_key=True)
    city_name = Column(String(100))
```

### 第 5 分钟：在代码中使用

#### 场景 1: 应用启动时自动同步

```python
# src/index.py
from src.database import engine, Base

@app.on_event("startup")
def init_database():
    Base.metadata.create_all(bind=engine)
    print("✅ 数据库已就绪")
```

#### 场景 2: 测试环境中使用

```python
# tests/conftest.py
@pytest.fixture(scope="function")
def test_db():
    Base.metadata.create_all(bind=engine)
    yield
    Base.metadata.drop_all(bind=engine)
```

## ❓ 常见问题（30秒解答）

### Q: 会删除我的数据吗？
**A**: 普通模式不会！只有使用 `--force` 参数才会删除。

### Q: 可以只创建某个表吗？
**A**: 可以！使用 `db_migration.py` 的选项 3。

### Q: 生产环境怎么用？
**A**: 建议使用 Alembic 进行版本化迁移。

### Q: 支持哪些数据库？
**A**: PostgreSQL（主要）、MySQL、SQLite。
