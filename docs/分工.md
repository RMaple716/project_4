# 两周极速开发四人分工方案（AI辅助+业务不熟适配版）
## 一、核心原则与优先级调整
### 🔴 绝对核心原则
1. **MVP优先**：只保留能跑通"用户输入→生成完整行程"的最小功能集，砍掉所有非必要功能
2. **AI深度赋能**：每个环节明确AI使用场景，用AI弥补业务不熟和时间不足的问题
3. **零交叉开发**：每个人负责完全独立的模块，避免沟通成本和依赖阻塞
4. **每日强制交付**：每天结束必须有可运行的成果，不允许"做了一半"的情况

### 🟠 模块优先级与功能裁剪
| 模块名称 | 优先级 | 保留功能 | 裁剪功能 |
|----------|--------|----------|----------|
| 用户交互模块 | 最高 | 表单输入+自然语言输入二选一<br>行程时间轴展示<br>基本增删改操作 | 拖拽调整、分享、导出PDF |
| 需求预处理模块 | 最高 | 关键词提取（目的地、天数、预算）<br>必填项缺失提醒 | 复杂意图识别、主动澄清对话 |
| 任务分解与分配模块 | 最高 | 固定任务分发逻辑<br>简单参数传递 | 动态任务调度、负载均衡 |
| 智能体独立规划模块 | 最高 | 景点智能体（已有）<br>交通智能体（已有）<br>住宿智能体（简化版）<br>美食智能体（简化版） | 复杂推荐算法、用户画像匹配 |
| 协商与整合模块 | 最高 | 时间冲突检测<br>简单路线优化<br>每日行程拼接 | 多智能体协商机制、复杂冲突解决 |
| 规划校验与输出模块 | 高 | 开放时间校验<br>总预算计算 | 交通可行性深度校验、异常处理 |
| 外部数据获取模块 | 高 | 天气API对接<br>景点基础数据 | 实时门票价格、导航API |
| 方案生成模块 | 高 | 前端渲染展示 | 个性化排版、多格式导出 |
| 管理员操作模块 | 最低 | **完全砍掉** | 所有功能 |

## 二、最终分工方案（完全独立+AI最大化）
### 成员A：前端全栈负责人（负责所有表现层）
**负责模块**：用户交互模块 + 方案生成模块
**核心任务**：
1. 第1-2天：用AI生成完整的前端项目模板（React/Vue），包含路由、状态管理
2. 第3-4天：开发用户需求输入页面（优先做表单输入，自然语言输入作为备选）
3. 第5-6天：开发行程展示页面（时间轴+每日详情）
4. 第7-8天：实现前端与后端的API对接
5. 第9-10天：开发行程编辑功能（增删景点、调整时间）
6. 第11-12天：优化页面样式和用户体验
7. 第13-14天：前端测试与bug修复

**AI辅助技巧**：
- 直接让AI生成"旅游规划系统前端页面代码"，包含需求输入和行程展示
- 用AI生成API调用代码，只需修改接口地址和参数
- 遇到样式问题，直接截图给AI让其生成CSS代码

### 成员B：需求与协调负责人（负责协调层核心）
**负责模块**：任务分解与分配模块 + 协商与整合模块
**核心任务**：
1. 第1-2天：用AI设计多智能体通信的数据格式（JSON），形成文档
2. 第3-4天：实现任务分解逻辑，将结构化需求拆分为各智能体的输入参数
3. 第5-6天：开发简单的时间冲突检测算法
4. 第7-8天：实现行程整合逻辑，将各智能体的输出拼接为每日行程
5. 第9-10天：开发基础的路线优化算法（避免折返）
6. 第11-12天：与成员C、D对接，调试智能体之间的通信
7. 第13-14天：协调层整体测试与bug修复

**AI辅助技巧**：
- 让AI生成"旅游行程时间冲突检测算法"和"简单路线优化算法"
- 用AI生成JSON数据格式示例和序列化/反序列化代码
- 遇到多智能体通信问题，让AI设计简单的消息传递机制

### 成员C：智能体开发负责人（负责业务逻辑层）
**负责模块**：需求预处理模块 + 智能体独立规划模块（住宿+美食）
**核心任务**：
1. 第1-2天：用AI开发简单的关键词提取功能，从用户输入中提取目的地、天数、预算
2. 第3-4天：完善已有的景点智能体，确保能正常输出景点列表和游览时间
3. 第5-6天：完善已有的交通智能体，确保能输出景点之间的交通方式和时间
4. 第7-8天：开发住宿智能体（简化版），根据预算和位置推荐酒店
5. 第9-10天：开发美食智能体（简化版），根据位置推荐当地特色餐厅
6. 第11-12天：统一所有智能体的输入输出格式
7. 第13-14天：智能体整体测试与bug修复

**AI辅助技巧**：
- 用AI生成"基于关键词的自然语言需求解析代码"
- 让AI生成住宿和美食推荐的基础逻辑（按价格、评分排序）
- 直接让AI修改已有的景点、交通智能体代码，统一接口格式

### 成员D：数据与后端负责人（负责数据层+后端服务）
**负责模块**：外部数据获取模块 + 规划校验与输出模块 + 后端服务搭建
**核心任务**：
1. 第1-2天：用AI生成后端项目模板（FastAPI/Django），搭建基础架构
2. 第3-4天：设计数据库表结构，存储景点、交通、住宿、美食基础数据
3. 第5-6天：对接天气API，获取目的地天气预报
4. 第7-8天：导入基础数据（可以用AI生成模拟数据，避免爬取）
5. 第9-10天：开发规划校验模块，检查景点开放时间和总预算
6. 第11-12天：编写所有API接口，供前端和其他模块调用
7. 第13-14天：后端整体测试与bug修复

**AI辅助技巧**：
- 让AI生成完整的后端项目结构和CRUD代码
- 用AI生成模拟数据（景点、酒店、餐厅），直接导入数据库
- 让AI生成天气API对接代码，只需替换API密钥

## 四、AI辅助开发关键技巧（必须掌握）
1. **代码生成**：直接告诉AI"用Python写一个旅游住宿推荐函数，输入是目的地、预算、位置，输出是酒店列表"
2. **问题解决**：遇到报错直接复制错误信息给AI，让其分析原因并给出解决方案
3. **文档生成**：让AI根据代码自动生成接口文档和数据格式文档
4. **测试用例**：让AI为每个模块生成单元测试用例
5. **数据生成**：让AI生成模拟的景点、酒店、餐厅数据，格式为SQL或JSON

## 五、风险应对措施
1. **某模块延期**：立即将该模块的非核心功能砍掉，只保留最基础的部分
2. **API对接失败**：直接使用AI生成的模拟数据，不影响整体流程
3. **算法效果不好**：先用最简单的规则实现，不追求完美
4. **集成问题**：每天下午预留1小时进行集成测试，及时发现问题
