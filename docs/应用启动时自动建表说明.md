# 🚀 应用启动时自动建表功能

## 📖 概述

项目已实现**应用启动时自动建表**功能，类似于 Sequelize 的 `sync()` 方法。当后端服务启动时，会自动检查并创建所有缺失的数据库表。

---

## ✨ 核心特性

### 1. 自动化
- ✅ 服务启动时自动执行
- ✅ 无需手动运行同步脚本
- ✅ 对开发者透明

### 2. 幂等性
- ✅ 多次启动不会重复创建表
- ✅ 安全调用，不会影响现有数据
- ✅ 可放心重启服务

### 3. 智能检测
- ✅ 自动识别缺失的表
- ✅ 只创建不存在的表
- ✅ 保留已有数据

### 4. 详细日志
- ✅ 显示建表过程
- ✅ 列出所有创建的表
- ✅ 错误信息清晰

## 🔧 实现方式

### 代码位置
文件：[`src/index.py`](file://d:\project\preoject_4\src\index.py)

### 关键代码

```python
# 应用启动事件：自动初始化数据库表
@app.on_event("startup")
def init_database():
    """
    应用启动时自动创建数据库表（类似 Sequelize sync）
    
    功能：
    - 检查所有模型定义的表是否存在
    - 自动创建缺失的表
    - 幂等操作，可安全多次调用
    """
    print("\n" + "="*80)
    print("🚀 正在初始化数据库...")
    print("="*80)
    
    try:
        # 自动创建所有表（如果不存在）
        Base.metadata.create_all(bind=engine)
        
        # 验证表是否创建成功
        from sqlalchemy import inspect
        inspector = inspect(engine)
        tables = inspector.get_table_names()
        
        print(f"\n✅ 数据库初始化完成！")
        print(f"📊 当前数据库共有 {len(tables)} 个表:")
        for table in sorted(tables):
            print(f"   - {table}")
        
        print("\n" + "="*80)
        print("✨ 数据库就绪，服务可以正常启动")
        print("="*80 + "\n")
        
    except Exception as e:
        print(f"\n❌ 数据库初始化失败: {e}")
        print("⚠️  请检查数据库连接配置是否正确")
        print("="*80 + "\n")
        raise
```

## ❓ 常见问题

### Q1: 如果数据库连接失败怎么办？

**现象**: 启动时显示错误信息

**解决**:
1. 检查 PostgreSQL 是否启动
2. 检查数据库连接配置
3. 检查数据库是否存在

```bash
# 测试数据库连接
psql -U postgres -d postgres
```

---

### Q2: 会删除我的数据吗？

**答案**: **不会！**

`Base.metadata.create_all()` 只会创建**不存在的表**，不会影响已有数据和表结构。

---

### Q3: 如何强制重建所有表？

**方法 1**: 使用同步脚本

```bash
python scripts/sync_db.py --force
```

**方法 2**: 手动删除后重启

```sql
-- 在数据库中执行
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
```

然后重启服务：

```bash
python src/index.py
```

---

### Q4: 生产环境也这样用吗？

**建议**: 

- ✅ **开发/测试环境**: 可以使用自动建表
- ⚠️ **生产环境**: 建议使用 **Alembic** 进行版本化迁移

**原因**:
- 生产环境需要更严格的变更控制
- Alembic 支持回滚和版本管理
- 避免意外修改表结构

---

### Q5: 能否禁用自动建表？

**方法**: 注释掉启动事件

```python
# 注释掉这段代码即可禁用
# @app.on_event("startup")
# def init_database():
#     ...
```

或者添加环境变量控制：

```python
import os

@app.on_event("startup")
def init_database():
    if os.getenv('AUTO_CREATE_TABLES', 'true').lower() != 'true':
        print("⚠️  自动建表已禁用")
        return
    
    # 正常的建表逻辑...
```

使用时：

```bash
# 禁用自动建表
AUTO_CREATE_TABLES=false python src/index.py
```

## 🎓 技术细节

### 核心 API

```python
from src.database import engine, Base

# 创建所有表（如果不存在）
Base.metadata.create_all(bind=engine)
```

### 工作原理

1. **检查元数据**: SQLAlchemy 读取所有模型的 `__tablename__`
2. **对比数据库**: 使用 `inspect` 检查数据库中已存在的表
3. **创建缺失表**: 只为不存在的表执行 `CREATE TABLE`
4. **创建索引和外键**: 自动处理所有约束

### 幂等性保证

```python
# 第一次调用
Base.metadata.create_all(bind=engine)  # 创建所有表

# 第二次调用
Base.metadata.create_all(bind=engine)  # 什么都不做（表已存在）

# 第三次调用
Base.metadata.create_all(bind=engine)  # 仍然什么都不做
```
