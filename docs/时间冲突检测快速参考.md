## 📡 API端点

### 1. 时间冲突检测
```
POST /api/v1/validation/time-conflict
```

**请求体**:
```json
{
  "schedule": [
    {
      "name": "活动名称",
      "start_time": "09:00",
      "end_time": "12:00",
      "activity_type": "attraction"
    }
  ]
}
```

**响应**:
```json
{
  "code": 200,
  "data": {
    "has_conflict": true/false,
    "conflicts": [...]
  }
}
```

### 2. 完整行程校验
```
POST /api/v1/validation/itinerary
```

**请求体**:
```json
{
  "day_plans": [...],
  "structured_requirement": {
    "total_budget": 5000
  }
}
```

## ⏰ 支持的时间格式

| 格式类型 | 示例 | 说明 |
|---------|------|------|
| 精确时间 | `"09:30"` | HH:MM格式 |
| 中文时间槽 | `"上午"` | 自动映射到9:00 |
| 英文时间槽 | `"morning"` | 自动映射到9:00 |
| 时长（小时） | `"2小时"` | 转换为120分钟 |
| 时长（范围） | `"2-3小时"` | 取平均150分钟 |
| 时长（分钟） | `"30分钟"` | 直接转换 |
| 特殊词汇 | `"半天"` | 转换为240分钟 |

## 🔍 冲突类型速查

| 类型 | 严重度 | 说明 |
|------|--------|------|
| `time_overlap` | error | 活动时间重叠 |
| `budget_exceeded` | error | 预算超支 |
| `unreasonable_time` | warning | 时间不合理（<6:00或>23:00） |
| `too_short_duration` | warning | 游览时间<30分钟 |
| `too_long_duration` | warning | 游览时间>8小时 |
| `overloaded_day` | warning | 单日>12小时 |

## 💡 Python调用示例

```python
import requests

# 检测时间冲突
response = requests.post(
    "http://127.0.0.1:9091/api/v1/validation/time-conflict",
    json={
        "schedule": [
            {"name": "景点A", "start_time": "09:00", "end_time": "11:00"},
            {"name": "午餐", "start_time": "10:30", "duration": "1小时"}
        ]
    }
)

result = response.json()
if result["data"]["has_conflict"]:
    print("发现冲突！")
    for c in result["data"]["conflicts"]:
        print(f"  - {c['description']}")
```

## ❓ 常见问题

**Q: 如何判断是否需要修复冲突？**  
A: 检查 `severity` 字段，`error` 必须修复，`warning` 可选优化。

**Q: 支持跨天活动吗？**  
A: 当前版本不支持，每个活动必须在同一天内完成。

**Q: 交通时间如何计算？**  
A: 目前使用简化估算（15-30分钟），后续将集成地图API。
