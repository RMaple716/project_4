## 📋 实际应用场景

### 场景1：智能体输出整合后的校验

当成员C的智能体返回结果后，成员B需要将这些结果整合成每日行程并校验：

```python
# 伪代码示例
def integrate_and_validate(agent_results):
    """整合智能体结果并校验"""
    
    # 1. 整合为每日行程
    day_plans = []
    for day in range(1, travel_days + 1):
        day_plan = {
            "day": day,
            "date": calculate_date(start_date, day),
            "attractions": agent_results["attraction"]["items_for_day"][day],
            "meals": agent_results["food"]["items_for_day"][day],
            "transport": agent_results["transport"]["routes_for_day"][day]
        }
        day_plans.append(day_plan)
    
    # 2. 调用校验接口
    response = requests.post(
        "http://127.0.0.1:9091/api/v1/validation/itinerary",
        json={
            "day_plans": day_plans,
            "structured_requirement": {
                "total_budget": 5000
            }
        }
    )
    
    result = response.json()
    
    # 3. 根据校验结果处理
    if result["data"]["valid"]:
        return {"status": "success", "itinerary": day_plans}
    else:
        # 有冲突，需要调整
        conflicts = result["data"]["conflicts"]
        suggestions = result["data"]["suggestions"]
        return {
            "status": "need_adjustment",
            "conflicts": conflicts,
            "suggestions": suggestions
        }
```

### 场景2：前端实时校验

用户在编辑行程时，前端可以实时调用校验接口：

```javascript
// React 示例
async function handleScheduleChange(newSchedule) {
  // 实时更新UI
  setSchedule(newSchedule);
  
  // 异步校验
  const response = await fetch('/api/v1/validation/time-conflict', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ schedule: newSchedule })
  });
  
  const result = await response.json();
  
  // 显示校验结果
  if (result.data.has_conflict) {
    setConflicts(result.data.conflicts);
    showNotification('发现时间冲突，请调整', 'warning');
  } else {
    setConflicts([]);
    showNotification('行程安排合理', 'success');
  }
}
```

def generate_optimization_suggestions(conflicts):
    """根据冲突生成优化建议"""
    suggestions = []

**输入**:

## 💡 最佳实践

### 1. 数据格式建议

**推荐**: 尽量使用精确时间格式
```json
{
  "start_time": "09:00",
  "end_time": "11:30"
}
```

**可接受**: 使用时间槽 + 时长
```json
{
  "start_time": "上午",
  "duration": "2小时"
}
```

### 2. 冲突处理策略

```python
# 优先级排序
severity_priority = {
    "error": 1,    # 必须修复
    "warning": 2   # 建议优化
}

# 先处理错误级别的冲突
errors = [c for c in conflicts if c["severity"] == "error"]
warnings = [c for c in conflicts if c["severity"] == "warning"]

if errors:
    # 阻止提交，要求用户修改
    return {"can_submit": False, "reasons": errors}
else:
    # 可以提交，但显示警告
    return {"can_submit": True, "warnings": warnings}
```

### 3. 性能优化

对于大量活动的检测，可以考虑：

```python
# 优化1: 按时间排序后再检测，减少比较次数
sorted_activities = sorted(activities, key=lambda x: x["start_minutes"])

# 优化2: 只比较相邻的活动（如果已排序）
for i in range(len(sorted_activities) - 1):
    act1 = sorted_activities[i]
    act2 = sorted_activities[i + 1]
    # 如果act1的结束时间 <= act2的开始时间，后续都不需要比较
    if act1["end_minutes"] <= act2["start_minutes"]:
        continue
```

## 🐛 常见问题排查

### 问题1: 服务启动失败

**症状**: `ModuleNotFoundError: No module named 'src.routes.validate'`

**解决**: 检查 `src/routes/__init__.py` 是否正确导出了 validate_router

### 问题2: 检测结果不准确

**症状**: 明明有时间重叠但未检测到

**排查步骤**:
1. 检查时间格式是否正确（必须是 "HH:MM" 或预定义时间槽）
2. 检查是否同时提供了 `end_time` 和 `duration`（优先使用 end_time）
3. 打印解析后的分钟数，确认转换正确

```python
# 调试代码
for item in schedule:
    start = parse_time_to_minutes(item.get("start_time"))
    end = parse_time_to_minutes(item.get("end_time")) if item.get("end_time") else None
    print(f"{item['name']}: {start} - {end}")
```

### 问题3: API 返回 422 错误

**原因**: 请求参数不符合 Pydantic 模型定义

**解决**: 检查请求体是否符合 `TimeConflictRequest` 或 `ItineraryValidateRequest` 的结构
