## 💡 实际应用场景

### 场景1: 完整的行程生成流程

```python
def generate_complete_itinerary(user_input):
    """从用户输入到完整行程的端到端流程"""
    
    # 1. 提交需求并获取结构化数据
    req_response = submit_requirement(user_input)
    structured_req = req_response["structured_requirement"]
    
    # 2. 任务分解
    task_response = decompose_task(structured_req)
    task_id = task_response["task_id"]
    
    # 3. 并行调用智能体
    agent_results = {}
    for subtask in task_response["subtasks"]:
        agent_type = subtask["agent_type"]
        result = call_agent(subtask)
        agent_results[agent_type] = result
    
    # 4. 整合行程
    integration_response = combine_itinerary(
        task_id, 
        agent_results, 
        structured_req
    )
    
    day_plans = integration_response["day_plans"]
    validation = integration_response["validation"]
    
    # 5. 根据校验结果处理
    if validation["valid"]:
        return {
            "status": "success",
            "itinerary": day_plans,
            "total_cost": integration_response["total_cost"]
        }
    else:
        return {
            "status": "need_adjustment",
            "itinerary": day_plans,
            "conflicts": validation["conflicts"],
            "suggestions": validation["suggestions"]
        }
```

### 场景2: 前端实时展示

```javascript
// 前端接收到整合结果后的处理
async function displayItinerary(integrationResult) {
  const { day_plans, validation, total_cost } = integrationResult;
  
  // 显示总花费
  document.getElementById('total-cost').textContent = `¥${total_cost}`;
  
  // 显示每天的行程
  day_plans.forEach(day => {
    const dayElement = createDayCard(day);
    document.getElementById('itinerary-container').appendChild(dayElement);
  });
  
  // 如果有冲突，显示警告
  if (!validation.valid) {
    validation.conflicts.forEach(conflict => {
      showWarning(conflict.description, conflict.severity);
    });
  }
  
  // 显示建议
  validation.suggestions.forEach(suggestion => {
    showInfo(suggestion);
  });
}
```

## 🔧 常见问题

### Q1: 整合后的行程为什么景点顺序不合理？

**A**: 当前使用的是简化版路线优化（按纬度排序）。如果需要更精确的路线规划，可以：
1. 调用 `/api/v1/integration/optimize-route` 手动优化
2. 集成地图API获取真实距离
3. 使用TSP算法进行路径规划

### Q2: 如何处理智能体返回的空数据？

**A**: 整合模块已实现容错处理：
- 空景点列表 → 当天无景点安排
- 空餐厅列表 → 不安排餐饮
- 空酒店列表 → 不添加住宿信息

建议在调用前检查智能体返回的状态：
```python
if agent_results["attraction"]["attractions"]:
    # 有景点数据，继续整合
else:
    # 使用默认景点或提示用户
```

### Q3: 校验发现冲突后如何调整？

**A**: 当前版本仅提供冲突检测和建议，不自动调整。你可以：
1. 根据 `suggestions` 手动调整行程
2. 修改用户需求后重新生成
3. （未来版本）实现自动调整算法

示例：
```python
if not validation["valid"]:
    for conflict in validation["conflicts"]:
        if conflict["type"] == "time_overlap":
            # 调整重叠的活动时间
            adjust_activity_time(conflict["activities"])
```

### Q4: 如何自定义餐饮安排？

**A**: 当前是自动安排三餐。如需自定义：
1. 在 `structured_requirement` 中添加餐饮偏好
2. 修改 `integrate_agent_results_to_daily_plans` 函数中的餐饮分配逻辑
3. 或者在整合后手动调整 `day_plans` 中的 `meals` 字段

## 📊 性能指标

### 当前性能
- **整合速度**: < 100ms (3天行程，6个景点)
- **内存占用**: < 5MB
- **并发支持**: 单实例支持10个并发请求

### 优化建议
对于大规模行程（>10天，>30个景点）：
1. 使用异步处理
2. 缓存常用路线
3. 分批整合（每天独立整合后合并）
