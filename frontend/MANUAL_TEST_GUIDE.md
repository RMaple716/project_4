# 🧪 需求表单手动测试指南

## 📋 测试前准备

### 1. 环境检查清单

在开始测试前，请确认：

- [ ] Python 3.7+ 已安装
- [ ] Node.js 已安装
- [ ] 后端依赖已安装（fastapi, uvicorn, pydantic等）
- [ ] 前端依赖已安装（运行过 `npm install`）

### 2. 启动服务

#### 方式一：使用自动化脚本（推荐）
```bash
cd frontend
run_test.bat
```

#### 方式二：手动启动

**终端1 - 启动后端：**
```bash
cd ..
python src/index.py
```
等待看到：`INFO: Uvicorn running on http://127.0.0.1:9091`

**终端2 - 启动前端：**
```bash
cd frontend
npm run dev
```
等待看到：`Local: http://localhost:3000/`

## 🎯 测试场景详解

### 场景1：基本功能测试（北京3日游）

#### 测试步骤

1. **访问页面**
   - 浏览器打开：`http://localhost:3000/requirement`
   - 确认页面正常加载，所有字段可见

2. **填写表单**
   ```

   出发日期：选择一个未来日期（如2026-06-15）
   旅行天数：3
   出行人数：2
   总预算：5000

   旅行偏好：历史古迹、美食探索、文化体验
   特殊需求：希望参观故宫和长城
   ```

3. **提交表单**
   - 点击"🚀 开始智能规划"按钮

4. **验证结果**
   - ✅ 显示"✅ 需求提交成功！"消息
   - ✅ 显示"正在智能规划行程..."加载提示
   - ✅ 自动跳转到 `/task/{task_id}` 页面

   - ✅ 每3秒更新一次进度
   - ✅ 完成后显示"🎉 任务分解成功！正在生成专属行程..."
   - ✅ 出现"查看行程"按钮
   - ✅ 点击按钮跳转到行程详情页

打开浏览器开发者工具（F12）→ Network标签，应该看到：

## 🔍 调试技巧

### 1. 查看控制台日志

**操作：**
1. 按 F12 打开开发者工具
2. 切换到 Console 标签
3. 提交表单

**应该看到：**
```javascript
提交需求: {city_name: "北京", travel_days: 3, ...}
```

---

### 2. 检查Network请求

**操作：**
1. F12 → Network标签
2. 勾选"Preserve log"
3. 提交表单

**检查点：**
- ✅ POST /api/v1/requirement/submit - Status 200
- ✅ POST /api/v1/task/decompose - Status 200
- ✅ Request Payload格式正确
- ✅ Response包含requirement_id和task_id

---

### 3. 查看Redux状态

**前提：** 安装Redux DevTools扩展

**操作：**
1. 打开Redux DevTools面板
2. 观察actions：
   - `requirement/setLoading` (true)
   - `requirement/setRequirement`
   - `requirement/setLoading` (false)

---

### 4. 检查路由跳转

**操作：**
1. 提交表单后观察地址栏
2. 应该从 `/requirement` 跳转到 `/task/{task_id}`

**验证：**
- ✅ task_id是有效的UUID格式
- ✅ 页面显示任务进度

## 🐛 常见问题排查

### 问题1：提交后没有反应

**可能原因：**
- 后端服务未启动
- API地址配置错误
- 网络连接问题

**解决方法：**
```bash
# 检查后端是否运行
netstat -ano | findstr :9091

# 重新启动后端
python src/index.py

# 检查浏览器控制台是否有错误
```

---

### 问题2：表单验证不生效

**可能原因：**
- rules配置错误
- Form.Item缺少name属性

**解决方法：**
1. 检查Form.Item的rules配置
2. 确认name属性与values对应
3. 查看Console是否有JavaScript错误

---

### 问题3：偏好标签不显示

**可能原因：**
- selectedPrefs状态未更新
- onChange事件未触发

**解决方法：**
1. 在onChange中添加console.log
2. 确认setSelectedPrefs被调用
3. 检查React DevTools中的组件状态

---

### 问题4：API返回404

**可能原因：**
- 后端路由未注册
- API路径错误

**解决方法：**
```bash
# 访问API文档确认接口存在
http://127.0.0.1:9091/docs

# 检查后端日志
```

---

### 问题5：TypeScript编译错误

**错误信息：**
```
找不到模块"antd"或其相应的类型声明
```

**解决方法：**
```bash
# 重新安装依赖
npm install

# 清除缓存后重装
npm cache clean --force
npm install
```
