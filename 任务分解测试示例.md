## 4. 预算分配算法说明

系统会根据总预算自动分摊（如果用户未指定具体预算）：
- **住宿**: 30% (1500元)
- **餐饮**: 25% (1250元)
- **交通**: 15% (750元)
- **门票**: 20% (1000元)
- **其他**: 10% (500元)

如果用户在结构化需求中指定了具体预算，则优先使用用户的值。

## 5. 业务规则验证

### 验证项
1. **出行天数**: 必须在 1-30 天之间
2. **出行人数**: 必须在 1-20 人之间
3. **最低预算**: 每人每天至少 100 元

```json
{
  "code": 400002,
  "msg": "预算过低，至少需要每人每天100元（最低300元）",

}
```

## 7. Python 测试脚本示例

```python
import requests
import json

BASE_URL = "http://127.0.0.1:9091"

def test_decompose():
    """测试任务分解"""
    url = f"{BASE_URL}/api/v1/task/decompose"
    
    payload = {
        "requirement_id": "req_test_001",
        "structured_requirement": {
            "city_name": "北京",
            "travel_days": 3,
            "total_budget": 5000,
            "travel_date": "2026-05-20",
            "traveler_count": 3,
            "preferences": ["历史古迹", "美食"],
            "dislikes": ["爬山"]
        }
    }
    
    response = requests.post(url, json=payload)
    result = response.json()
    
    print("任务分解结果:")
    print(json.dumps(result, indent=2, ensure_ascii=False))
    
    if result["code"] == 200:
        task_id = result["data"]["task_id"]
        print(f"\n任务ID: {task_id}")
        
        # 查询任务状态
        status_url = f"{BASE_URL}/api/v1/task/{task_id}"
        status_response = requests.get(status_url)
        print("\n任务状态:")
        print(json.dumps(status_response.json(), indent=2, ensure_ascii=False))
    
    return result

if __name__ == "__main__":
    test_decompose()
```

## 8. 注意事项

1. **任务ID关联**: 
   - `batch_id` 是主任务ID，用于查询整体进度
   - `subtask_id` 是子任务ID，用于更新各个智能体的结果

2. **状态流转**:
   - `pending`: 任务已创建，等待执行
   - `running`: 至少有一个子任务在执行
   - `success`: 所有子任务完成
   - `failed`: 有子任务失败

3. **进度计算**:
   - 主任务的进度 = (已完成子任务数 / 总子任务数) * 100
   - 前端可以轮询 `/api/v1/task/{task_id}` 获取实时进度

4. **智能体调用顺序**:
   - 4个子任务可以并行执行，没有依赖关系
   - 每个智能体完成后调用 `/api/v1/task/update/{subtask_id}` 更新结果
