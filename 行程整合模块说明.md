# 行程整合模块说明文档

## 📋 功能概述

行程整合模块负责将各智能体（景点、住宿、美食、交通）的输出结果拼接为完整的每日行程，并进行时间冲突检测。

**核心职责**：
- 接收4个智能体的输出数据
- 按天数分配景点和活动
- 安排餐饮和交通
- 计算每日花费
- 调用校验接口检测时间冲突

## 🧠 核心算法

### 1. 景点分配算法

**策略**: 均匀分配到每天

```python
# 计算每天分配的景点数
attractions_per_day = len(optimized_attractions) // travel_days

# 按顺序分配
for day in range(1, travel_days + 1):
    day_attractions = attractions[day * attractions_per_day : (day+1) * attractions_per_day]
```

### 2. 时间槽映射

将智能体返回的 `visit_time_slot` 转换为具体时间段：

| visit_time_slot | 中文时段 | 开始时间 | 结束时间 |
|----------------|---------|---------|---------|
| morning | 上午 | 09:00 | 12:00 |
| afternoon | 下午 | 14:00 | 17:00 |
| evening | 晚上 | 18:00 | 20:00 |

### 3. 餐饮安排

每天自动安排三餐：
- **早餐**: 07:30
- **午餐**: 12:30
- **晚餐**: 18:30

从餐厅列表中循环选择，确保每天都有不同的餐厅推荐。

### 4. 交通估算

简化版交通时间估算：
- 同一区域或近距离: **15分钟**
- 不同区域: **30分钟**
- 默认交通费: **5元/人**

### 5. 路线优化

当前实现：按纬度排序（简化版）

```python
sorted_attractions = sorted(
    attractions, 
    key=lambda x: x.get("location", {}).get("lat", 0)
)
```

**未来改进方向**:
- 使用TSP（旅行商问题）算法
- 集成地图API获取真实距离
- 考虑交通方式和路况

## 📊 数据结构

### 每日行程单元 (DayPlan)

```python
{
    "day": int,                    # 第几天（从1开始）
    "date": str,                   # 日期 YYYY-MM-DD
    "attractions": List[Dict],     # 当天景点列表
    "meals": List[Dict],           # 当天餐饮列表
    "transport": Dict | None,      # 交通信息
    "hotel": Dict | None,          # 住宿信息（仅第一天）
    "daily_cost": float,           # 当日总花费
    "notes": str                   # 备注
}
```

### 景点数据结构

```python
{
    "id": str,
    "name": str,
    "category": str,
    "visit_time_slot": str,        # morning/afternoon/evening
    "visit_time": str,             # 上午/下午/晚上（中文）
    "start_time": str,             # HH:MM
    "end_time": str,               # HH:MM
    "ticket_price": float,
    "location": {"lat": float, "lng": float},
    "tags": List[str]
}
```

### 餐饮数据结构

```python
{
    "id": str,
    "name": str,
    "cuisine": str,
    "meal_type": str,              # breakfast/lunch/dinner
    "meal_time": str,              # 早上/中午/晚上
    "time": str,                   # HH:MM
    "avg_price_per_person": float,
    "rating": float
}
```

## 🔗 与其他模块的集成

### 1. 与任务分解模块 (task.py)

```python
# 流程示例
# 1. 任务分解
subtasks = decompose_to_subtasks(requirement_id, structured_req)

# 2. 调用各智能体获取结果
agent_results = {
    "attraction": call_attraction_agent(subtasks[0]),
```

### 2. 与校验模块 (validate.py)

整合模块内部自动调用校验接口：


如果检测到冲突，会在响应中返回：
- `conflicts`: 冲突列表
- `suggestions`: 调整建议

## ⚠️ 注意事项

### 1. 数据完整性

确保智能体返回的数据包含必要字段：
- 景点必须有 `location`（用于路线优化）
- 景点必须有 `visit_time_slot`（用于时间安排）
- 餐厅必须有 `avg_price_per_person`（用于预算计算）

### 2. 容错处理

模块已实现以下容错机制：
- 空数据处理：如果某类智能体返回空列表，跳过该项
- 缺失字段：使用默认值填充
- 异常捕获：所有错误都会返回友好的错误消息

### 3. 性能考虑

当前实现为简化版：
- 路线优化使用简单排序（O(n log n)）
- 交通时间使用固定估算值
- 适合小规模行程（≤10天，≤30个景点）

**如需处理大规模行程，建议**:
- 使用更复杂的路径规划算法
- 缓存常用路线的交通时间
- 异步处理整合逻辑

## 🚀 后续优化方向

   - 根据景点位置就近推荐餐厅

### 中期优化（1个月）
1. **多日均衡性**
   - 避免某天过累，某天过闲

2. **个性化调整**
   - 支持用户手动调整行程顺序
   - 记住用户的历史偏好

3. **天气适配**
   - 雨天自动减少户外活动
   - 高温天增加室内景点

### 长期优化（3个月）

2. **社交化功能**

3. **实时数据集成**
